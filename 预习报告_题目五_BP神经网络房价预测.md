## 一、课题内容与分析

### 1.1 课题背景

波士顿房价数据集（Boston Housing Dataset）是机器学习领域经典的回归分析数据集，记录了美国波士顿地区 506 个街区的 13 项社会经济特征与对应房屋的中位价格（单位：千美元）。该数据集广泛用于验证回归算法的预测性能，是学习 BP 神经网络的理想案例。

### 1.2 系统目标

本课题旨在利用 Python 语言设计并实现一个基于 BP 神经网络的房价预测系统，完成以下目标：

- 读取并预处理波士顿房价数据集，包括缺失值检查、异常值处理、数据标准化
- 构建并训练 BP 神经网络回归模型，实现房价预测
- 对模型性能进行全面评估（MAE、MSE、RMSE、R² 等指标）
- 通过可视化展示训练过程、预测效果和误差分布
- 提供交互式命令行界面，支持用户输入特征进行单条房价预测
- 支持不同超参数（Epoch 数量）对模型性能影响的对比分析

### 1.3 关键问题分析

1. **数据预处理**：原始数据各特征量纲差异大（如 CRIM 范围为 0~100，CHAS 仅为 0/1），需进行标准化处理；同时需检测和处理可能的异常值。
2. **神经网络结构设计**：需合理选择隐藏层层数、神经元数量、激活函数、学习率等超参数。
3. **过拟合防止**：BP 神经网络在训练过程中容易过拟合，需引入 L2 正则化和早停机制。
4. **预测结果的可靠性**：需使用多种评估指标综合评价模型性能。

## 二、课题相关基础知识

### 2.1 BP 神经网络基本原理

BP（Back Propagation，反向传播）神经网络是 1986 年由 Rumelhart、Hinton 和 Williams 提出的一种多层前馈神经网络，是人工神经网络中最经典、应用最广泛的模型之一。其核心思想是利用梯度下降法，通过误差反向传播来逐层调整网络的连接权值和阈值，从而使网络的输出不断逼近期望输出。

### 2.2 BP 神经网络的结构

BP 神经网络由三个基本部分组成：**输入层（Input Layer）**、**隐藏层（Hidden Layer）** 和 **输出层（Output Layer）** 。

**（1）输入层**：负责接收外部输入数据。在波士顿房价预测任务中，输入层包含 13 个神经元，分别对应 CRIM、ZN、INDUS、CHAS、NOX、RM、AGE、DIS、RAD、TAX、PIRATIO、B、LSTAT 这 13 个特征。输入层神经元仅负责数据传递，不进行计算处理。

**（2）隐藏层**：位于输入层和输出层之间，是 BP 网络的核心计算部分。隐藏层可以有一层或多层，每层包含若干个神经元。本系统采用三层隐藏层结构，神经元数量分别为 64、32、16，逐层递减以提取特征的高阶抽象表示。隐藏层的层数和神经元数量是网络的重要超参数，直接影响模型的表达能力和泛化性能。

**（3）输出层**：输出网络的最终计算结果。在房价预测任务中，输出层仅包含 1 个神经元，输出值代表预测的房价（千美元）。

各层之间的神经元通过**连接权值（Weight）** 实现全连接，即每个神经元与相邻层的所有神经元相连，每个连接对应一个权重值。此外，每个神经元（除输入层外）还带有一个**偏置项（Bias）**。

### 2.3 BP 算法的数学原理

#### 2.3.1 前向传播过程

前向传播是指信号从输入层经过隐藏层逐层处理到达输出层的过程。以单层隐藏层为例，设输入向量为 X = (x1, x2, ..., xn)，隐藏层的权值矩阵为 W1，偏置向量为 b1，激活函数为 f，则隐藏层的输出 H 为：

    H = f(W1 * X + b1)

输出层的输出 Y_pred 为：

    Y_pred = W2 * H + b2

其中 W2 为输出层权值矩阵，b2 为输出层偏置。对于多层隐藏层，上述过程逐层递推执行。

#### 2.3.2 损失函数

在回归任务中，常用的损失函数为**均方误差（Mean Squared Error, MSE）**：

    Loss = (1/m) * Σ(y_true_i - y_pred_i)^2

其中 m 为样本数量，y_true_i 为第 i 个样本的真实房价，y_pred_i 为网络预测值。损失函数的值越小，表示网络预测越准确。

#### 2.3.3 反向传播过程

反向传播是 BP 算法的核心，其目标是使损失函数值最小化。具体步骤如下：

**步骤一**：计算输出层误差。输出层第 j 个神经元的误差 δ_output_j 为损失函数对输出层净输入的偏导数。

**步骤二**：反向传播误差。隐藏层第 i 个神经元的误差 δ_hidden_i 由输出层误差加权求和后再乘以激活函数的导数得到。

**步骤三**：更新权值和偏置。权值更新公式为：

    W_new = W_old - learning_rate * (∂Loss / ∂W)

偏置更新公式为：

    b_new = b_old - learning_rate * (∂Loss / ∂b)

其中 learning_rate（学习率）控制每次更新的步长，是重要的超参数。上述更新过程基于**梯度下降法（Gradient Descent）**，即沿损失函数梯度的反方向调整参数，使损失值逐步下降。

#### 2.3.4 链式法则

反向传播的数学基础是微积分中的**链式法则（Chain Rule）**。通过链式法则，可以将输出层误差逐层分解到每个权值和偏置上，从而计算出各参数对总误差的贡献度（梯度），实现高效的参数更新。

### 2.4 激活函数

激活函数为神经网络引入非线性变换能力，使网络能够学习和逼近任意复杂的非线性映射关系。常用的激活函数包括：

**（1）Sigmoid 函数**：

    f(x) = 1 / (1 + e^(-x))

输出范围 (0, 1)，适用于二分类输出层。存在梯度饱和问题，在深层网络中容易导致梯度消失。

**（2）Tanh 函数**：

    f(x) = (e^x - e^(-x)) / (e^x + e^(-x))

输出范围 (-1, 1)，均值为 0，比 Sigmoid 收敛更快，但仍存在梯度饱和问题。

**（3）ReLU 函数（Rectified Linear Unit）**：

    f(x) = max(0, x)

计算简单，能有效缓解梯度消失问题，训练速度更快，是目前最广泛使用的激活函数。本系统的隐藏层采用 ReLU 激活函数。

**（4）线性激活函数**：

    f(x) = x

用于回归任务的输出层，输出值不受限制，可直接作为预测结果。本系统的输出层采用线性激活函数。

### 2.5 优化算法

BP 神经网络的训练过程本质上是优化问题。常用的优化算法包括：

**（1）标准梯度下降（SGD）**：每次更新使用全部训练样本计算梯度，收敛稳定但速度慢。

**（2）Adam 优化器（Adaptive Moment Estimation）**：结合动量法和自适应学习率，能根据每个参数的历史梯度信息动态调整学习率，收敛速度快且稳定性好。本系统的默认优化器采用 Adam。

### 2.6 防止过拟合技术

**（1）L2 正则化（权值衰减）**：在损失函数中加入权值的平方和项，惩罚过大的权值，防止网络对训练数据过度拟合。

    Loss_reg = Loss + (alpha / 2) * Σ(W^2)

**（2）早停机制（Early Stopping）**：在训练过程中监控验证集上的损失值，若连续多个轮次（如 10 轮）验证集损失未下降，则提前终止训练，防止过拟合并节省训练时间。

### 2.7 关键技术工具

**Python 编程语言与库**：系统使用 Python 3 开发，依赖以下主要库：

- **scikit-learn**：提供 `MLPRegressor`（多层感知机回归器）实现 BP 神经网络，以及 `train_test_split`（数据划分）、`StandardScaler`（标准化）、评估指标计算函数等
- **pandas**：数据加载（read_csv）、清洗、统计分析
- **numpy**：数值计算与矩阵运算
- **matplotlib**：数据可视化，绘制 Loss 曲线、预测对比散点图、残差分布图等

**模型评估指标**：

$$MAE = \frac{1}{n}\sum_{i=1}^{n} |y_i - \hat{y}_i|$$

$$MSE = \frac{1}{n}\sum_{i=1}^{n} (y_i - \hat{y}_i)^2$$

$$RMSE = \sqrt{MSE}$$

$$R^2 = 1 - \frac{\sum_{i=1}^{n} (y_i - \hat{y}_i)^2}{\sum_{i=1}^{n} (y_i - \bar{y})^2}$$

### 2.8 开发环境

项目使用 conda 虚拟环境（pigliu），集成 numpy 2.0.2、pandas 2.2.3、scikit-learn 1.6.1、matplotlib 3.10.0。代码采用模块化设计，各功能独立封装于单独的 .py 文件中，结构清晰，便于维护和扩展。

## 三、课题设计思路

### 3.1 总体方案

本系统采用命令行菜单交互方式，用户通过数字选择功能模块执行对应操作。系统按照 **数据加载 → 数据预处理 → 模型训练 → 模型评估 → 预测 → 可视化** 的顺序组织主体流程。

### 3.2 功能模块划分

| 模块 | 功能描述 |
|------|----------|
| 数据加载模块 | 从本地 CSV 文件读取波士顿房价数据，共 506 条样本、13 个特征 |
| 数据清洗模块 | 缺失值检查与处理、异常值检测（3-sigma 原则） |
| 数据归一化模块 | 采用 StandardScaler 进行 Z-Score 标准化 |
| 模型构建模块 | 搭建三层隐藏层 BP 网络，配置超参数 |
| 模型训练模块 | 训练网络、保存模型文件、记录训练历史 |
| 模型评估模块 | 计算 MAE/MSE/RMSE/R²/MAPE 五项指标 |
| 房价预测模块 | 支持批量预测与单条用户输入预测 |
| 可视化模块 | 绘制 Loss 曲线、预测对比散点图、残差图、特征分布图 |
| 超参数对比模块 | 比较不同 Epoch 对模型性能的影响 |

### 3.3 输入输出设计

- **输入**：13 个房屋特征数值（CRIM, ZN, INDUS, CHAS, NOX, RM, AGE, DIS, RAD, TAX, PIRATIO, B, LSTAT）
- **处理**：标准化 → BP 神经网络前向传播 → 反标准化还原房价
- **输出**：预测房价（千美元）、评估指标、可视化图表

### 3.4 程序运行流程

启动程序 → 显示主菜单 → 用户选择功能 → 执行对应操作 → 返回菜单

推荐操作顺序：加载数据 → 训练模型 → 评估模型 → 进行预测 → 查看可视化
