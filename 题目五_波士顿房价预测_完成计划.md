## 题目五：基于 BP 神经网络的波士顿房价预测系统设计与实现 - 完成计划

### 题目概述

利用 Python 语言设计并实现一个基于 BP 神经网络的房价预测系统，读取波士顿房价数据集，对数据进行预处理，构建 BP 神经网络模型，实现房价预测，并对模型性能进行分析和可视化展示。

### 题目要求（必须满足）

1. 使用 Python 语言编程
2. 程序具有良好的交互界面
3. 对输入数据进行异常处理，并提示错误信息
4. 程序要求添加必要注释

### 开发计划

#### 第一阶段：项目环境与框架搭建

| 任务 | 内容 | 预计产出 |
|------|------|----------|
| 1.1 | 安装依赖库：`numpy`, `pandas`, `matplotlib`, `scikit-learn` | `requirements.txt` |
| 1.2 | 创建主程序入口 `main.py`，搭建菜单式或简单 GUI 交互界面 | 程序主框架 |
| 1.3 | 设计代码模块结构，明确各模块职责 | 模块划分方案 |

**技术选型说明**：BP 神经网络可以用 `scikit-learn` 的 `MLPRegressor`（推荐，标准化接口），也可以用 `numpy` 手写实现（展示理解深度）。建议先用 `MLPRegressor` 快速完成功能，如有余力再手写一版对比。

#### 第二阶段：数据管理（功能要求一）

| 任务 | 内容 | 预计产出 |
|------|------|----------|
| 2.1 | 加载波士顿房价数据集（`sklearn.datasets.load_boston` 已废弃，改用 `fetch_california_housing` 或从 CSV 文件加载） | `data_loader.py` |
| 2.2 | 显示数据基本信息：样本数、特征数、特征名称、前 5 行预览 | `data_info()` 函数 |
| 2.3 | 缺失值检查与处理（均值填补或删除） | `data_clean()` 函数 |
| 2.4 | 异常值检测（箱线图法 / 3σ 原则）与处理 | 同上 |
| 2.5 | Min-Max 归一化或 Z-Score 标准化 | `data_normalize()` 函数 |

**注意事项**：
- `sklearn.datasets.load_boston` 在 1.2 版本已因伦理问题移除。替代方案：从 `sklearn.datasets.fetch_california_housing` 获取加州房价数据，或要求用户提供 CSV 数据文件。
- 归一化必须先用训练集的参数（min/max 或 mean/std），再作用于测试集，避免数据泄漏。

#### 第三阶段：BP 神经网络模型构建与训练（功能要求二）

| 任务 | 内容 | 预计产出 |
|------|------|----------|
| 3.1 | 划分训练集和测试集（建议 8:2），设置随机种子保证可复现 | `train_test_split` |
| 3.2 | 构建 BP 神经网络：确定输入层（13 特征）、隐藏层（1-3 层，节点数如 64→32→16）、输出层（1 节点） | `model_builder.py` |
| 3.3 | 配置超参数：学习率、最大迭代次数、激活函数（ReLU）、优化器（Adam/SGD） | 参数配置 |
| 3.4 | 训练模型并保存到文件（`joblib` 或 `pickle`） | `model_train.py` |
| 3.5 | 训练过程中实时显示 Loss 变化 | `plot_loss()` |

**BP 网络推荐结构**：
```
Input(13) → Dense(64, ReLU) → Dense(32, ReLU) → Dense(16, ReLU) → Output(1)
```

#### 第六阶段：数据可视化（功能要求五）

### 关键技术难点预判

| 难点 | 解决方案 |
|------|----------|
| Boston 数据集无法直接 import | 使用 `fetch_california_housing` 替代，或从网盘链接下载 CSV 直接读取 |
| BP 网络训练不收敛 | 调整学习率（从 0.001 起步），增加隐藏层节点数，检查数据归一化是否到位 |
| 过拟合 | 增加 `alpha`（L2 正则化参数），调小隐藏层规模，使用 early stopping |
| 用户输入预测时的归一化/反归一化 | 归一化参数（min/max）必须从训练集计算并保存，预测时复用同一参数 |
| 可视化中文乱码 | 设置 `plt.rcParams['font.sans-serif'] = ['SimHei']` 或使用英文标签 |

### 测试计划

| 测试场景 | 验证内容 |
|----------|----------|
| 数据加载 | 正确读取数据集，特征数=13，样本数符合预期 |
| 数据清洗 | 缺失值正确处理、异常值被过滤或修正 |
| 归一化 | 所有特征落入 [0,1] 或均值为 0 方差为 1 的区间 |
| 模型训练 | Loss 随 Epoch 递减，最终收敛 |
| 模型保存/加载 | 保存的模型可被加载并输出一致预测结果 |
| 单条预测 | 输入合法 13 个特征值，输出合理房价数值 |
| 异常输入 | 非数字、空值、越界数值 → 提示错误信息 |
| 评估指标 | MAE < 5（加州数据: < 0.5），R² > 0.6 |
| 可视化 | 图表正确显示，不崩溃，中文无乱码 |
| Epoch 对比 | 不同 Epoch 下指标有明显趋势变化 |
